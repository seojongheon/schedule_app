# 주간 캘린더 일정 슬롯 오버플로 개선 설계

## 목표

주간 캘린더의 하루 미리보기에서 긴 일정 제목과 더보기 문구가 날짜 셀의 옆 칸을 침범하거나 다음 줄로 내려가지 않도록 한다. 현재 표시 정책인 일정 2개와 초과 일정 더보기 슬롯은 유지한다.

## 현재 동작과 원인

- 주간 캘린더는 7열 그리드 안에서 날짜별 일정을 최대 2개 표시한다.
- 일정이 3개 이상이면 `+N개` 더보기 문구를 추가한다.
- 일정 제목 자체에는 `truncate`가 적용되어 있지만, 그리드 아이템인 날짜 셀과 그 안의 일정 컨테이너에 축소 가능한 최소 너비와 오버플로 경계가 모두 명시되어 있지 않다.
- 따라서 긴 한글, 긴 영문, 공백 없는 문자열처럼 고유 너비가 큰 제목은 좁은 모바일 화면에서 날짜 셀의 계산 너비보다 콘텐츠 영역을 크게 만들 수 있다.
- 슬롯 높이와 한 줄 표시 규칙을 컨테이너 계층 전체에서 고정하지 않아 브라우저와 글꼴 조건에 따라 줄바꿈 또는 세로 밀림이 발생할 여지가 있다.

## 선택한 접근

`ScheduleCalendar`의 현재 구조와 표시 개수 로직은 유지하고, 주간 날짜 셀부터 개별 슬롯까지 각 레이아웃 경계에 너비 축소와 오버플로 제한을 명시한다.

- 날짜 셀: 7열 그리드의 할당 폭보다 커지지 않도록 `min-width: 0`, `max-width: 100%`, `overflow: hidden`을 적용한다.
- 일정 목록: 부모 폭을 그대로 따르며 자식의 고유 너비 때문에 확장되지 않도록 동일한 축소 경계를 적용한다.
- 일정 슬롯: 고정 높이, 한 줄, 가로 오버플로 숨김, 말줄임을 명시한다.
- 더보기 슬롯: 일정 슬롯과 동일한 높이 및 한 줄 경계를 사용하되 기존의 중립 색상과 `+N개` 문구는 유지한다.

별도 슬롯 컴포넌트 분리나 반응형 글자 크기 축소는 이번 문제 해결에 필요하지 않으므로 포함하지 않는다.

## 표시 규칙

- 일정 0개: 슬롯을 표시하지 않는다.
- 일정 1개: 일정 슬롯 1개를 표시한다.
- 일정 2개: 일정 슬롯 2개를 표시한다.
- 일정 3개 이상: 앞의 일정 슬롯 2개와 `+{전체 개수 - 2}개` 더보기 슬롯 1개를 표시한다.
- 모든 일정 제목과 더보기 문구는 한 줄로만 표시한다.
- 제목이 슬롯 너비를 넘으면 말줄임표로 생략하고 옆 날짜 셀은 침범하지 않는다.
- 날짜 셀의 전체 높이는 콘텐츠 길이에 따라 늘어나거나 줄바꿈으로 밀리지 않는다.

## 코드 구조

- `src/components/app/ScheduleCalendar.tsx`
  - 순수 표시 로직을 사용하고 기존 `renderWeekDays` 렌더링에 너비와 오버플로 제약을 적용한다.
- `src/components/app/week-calendar-preview.ts`
  - 일정 배열을 앞의 2개와 초과 개수로 나누는 순수 함수를 제공한다.
- `src/components/app/week-calendar-preview.test.mjs`
  - Node 내장 테스트 러너로 순수 표시 로직을 검증한다.
- `package.json`
  - Node 24의 타입 제거 기능을 사용해 새 외부 의존성 없이 해당 테스트를 실행하는 `test:unit` 명령을 추가한다.

## 테스트 전략

### 자동 테스트

- 0, 1, 2개의 일정은 모두 보이고 더보기 수가 0인지 검증한다.
- 3개의 일정은 2개만 보이고 더보기 수가 1인지 검증한다.
- 다수 일정에서도 표시 개수는 2개로 고정되고 초과 개수가 정확한지 검증한다.
- 실패 테스트를 먼저 실행해 기존 코드에 테스트 대상 경계가 없거나 요구 동작을 만족하지 못함을 확인한 후 구현한다.

### 정적 검증

- TypeScript 타입 검사를 통과해야 한다.
- ESLint 검사를 통과해야 한다.
- Next.js 프로덕션 빌드를 통과해야 한다.

### 브라우저 검증

주간 보기를 320px, 375px, 430px 너비에서 확인한다.

- 짧은 제목
- 긴 한글 제목
- 긴 영문 제목
- 공백 없는 매우 긴 문자열
- 일정 0개, 1개, 2개, 3개, 다수
- 이전 주와 다음 주 슬라이드 화면

각 경우에 제목이 한 줄 말줄임으로 끝나고, 옆 날짜 셀 침범이나 슬롯 아래 밀림이 없으며, 3개 이상일 때 기존 더보기 슬롯이 유지되는지 확인한다.

## 변경 범위와 보호 사항

- 월간 캘린더와 시간표 일정 블록의 표시 방식은 변경하지 않는다.
- 검색, 참가자 필터, 주간 스와이프, 날짜 선택 동작은 변경하지 않는다.
- 더보기 슬롯에 새로운 클릭 동작을 추가하지 않는다. 날짜 셀 선택 동작을 그대로 따른다.
- 작업 시작 전에 존재하던 `src/components/app/ScheduleWorkspace.tsx`의 미커밋 변경은 수정하거나 커밋하지 않는다.
- 외부 패키지는 추가하지 않는 것을 기본 원칙으로 한다.
