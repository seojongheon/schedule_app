# Quickstart Validation: 주간 캘린더 일정 슬롯 오버플로 개선

## Prerequisites

- Node.js 24.x
- `npm ci` 완료
- 로컬 실행에 필요한 환경 변수 설정

## Automated Verification

```bash
npm run test:unit
npm run typecheck
npm run lint
npm run build
```

Expected outcomes:

- 단위 테스트는 0, 1, 2, 3, 8, 100개 일정의 표시/초과 개수 규칙을 모두 통과한다.
- 타입 검사와 린트는 오류 없이 종료한다.
- 프로덕션 빌드는 종료 코드 0으로 완료한다.

## Browser Validation

1. `npm run dev`로 애플리케이션을 실행한다.
2. 주간 보기에서 한 날짜에 긴 한글 제목, 긴 영문 제목, 공백 없는 긴 문자열과 총 3개 이상의 일정을 준비한다.
3. 320px, 375px, 430px 뷰포트에서 각각 현재 주를 확인한다.
4. 각 일정 제목이 한 줄 말줄임이고 인접 날짜 칸을 침범하지 않는지 확인한다.
5. 일정 2개와 정확한 `+N개` 더보기 슬롯이 동일한 날짜 칸 안에 표시되는지 확인한다.
6. 이전 주와 다음 주로 이동해 전환 중 및 전환 후에도 같은 경계가 유지되는지 확인한다.
7. 검색 및 참가자 필터를 적용해 강조도만 바뀌고 슬롯 크기가 유지되는지 확인한다.
8. 날짜를 선택해 시간표가 기존처럼 바뀌는지 확인한다.
9. 월간 보기로 전환해 월간 일정 슬롯이 변경되지 않았는지 확인한다.

## Computed Layout Checks

각 뷰포트에서 긴 제목 슬롯과 더보기 슬롯에 대해 다음을 확인한다.

- 슬롯의 `scrollWidth`가 날짜 셀의 `clientWidth`를 넘더라도 슬롯 자체의 바깥 경계는 날짜 셀의 경계 안에 있다.
- 슬롯의 계산된 `white-space` 값이 한 줄 표시를 강제한다.
- 첫 일정, 두 번째 일정, 더보기 슬롯의 높이가 제목 길이에 따라 변하지 않는다.
- 날짜 셀의 오른쪽 경계가 다음 날짜 셀의 왼쪽 경계를 넘지 않는다.
