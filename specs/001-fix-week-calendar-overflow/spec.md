# Feature Specification: 주간 캘린더 일정 슬롯 오버플로 개선

**Feature Branch**: `codex/week-calendar-overflow`

**Created**: 2026-07-18

**Status**: Ready for Planning

**Input**: User description: "주간 캘린더의 일정 제목이 잘리거나 옆 칸을 침범하고 아랫줄로 밀리는 문제를 찾아 수정한다. 일정은 기존처럼 2개만 보여주고 초과분은 더보기로 표시하며 수정 후 테스트한다."

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 좁은 화면에서도 일정 제목을 안정적으로 확인 (Priority: P1)

사용자는 모바일 주간 캘린더에서 일정 제목의 길이와 문자 구성에 관계없이 각 일정이 해당 날짜 칸 안에 한 줄로 정리되어 표시되는 것을 본다. 제목 전체가 들어가지 않으면 해당 칸 안에서 생략되며 옆 날짜나 아래 슬롯의 배치를 깨뜨리지 않는다.

**Why this priority**: 주간 캘린더의 날짜별 일정을 빠르게 훑는 핵심 사용 흐름이며, 현재 문제는 인접 날짜의 정보까지 읽기 어렵게 만든다.

**Independent Test**: 320px, 375px, 430px 너비의 주간 캘린더에 긴 한글 제목, 긴 영문 제목, 공백 없는 문자열을 각각 표시하고 모든 제목이 자기 날짜 칸의 한 줄 안에 머무는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 320px 너비의 주간 캘린더와 날짜 칸보다 긴 한글 일정 제목이 있을 때, **When** 사용자가 해당 주를 확인하면, **Then** 제목은 한 줄로 생략되고 인접 날짜 칸을 침범하지 않는다.
2. **Given** 공백 없는 긴 영문 또는 숫자 문자열 제목이 있을 때, **When** 사용자가 375px 또는 430px 너비에서 해당 주를 확인하면, **Then** 문자열은 날짜 칸 경계에서 생략되고 두 번째 줄을 만들지 않는다.
3. **Given** 사용자가 이전 주나 다음 주로 이동할 때, **When** 전환 중과 전환 후의 주간 미리보기가 표시되면, **Then** 세 주 패널 모두 동일한 제목 경계 규칙을 유지한다.

---

### User Story 2 - 기존 일정 개수와 더보기 규칙 유지 (Priority: P2)

사용자는 하루에 일정이 많아도 주간 미리보기에서 앞의 일정 2개와 초과 개수를 나타내는 더보기 슬롯을 기존과 같은 방식으로 본다.

**Why this priority**: 오버플로 수정 과정에서 이미 익숙한 정보 밀도와 날짜 선택 흐름이 바뀌지 않아야 한다.

**Independent Test**: 한 날짜에 일정 수를 0, 1, 2, 3, 8개로 바꾸어 표시 일정 수와 더보기 개수가 각각 규칙에 맞는지 확인한다.

**Acceptance Scenarios**:

1. **Given** 한 날짜에 일정이 0개, 1개 또는 2개 있을 때, **When** 주간 미리보기를 확인하면, **Then** 존재하는 일정만 최대 2개 표시되고 더보기 슬롯은 나타나지 않는다.
2. **Given** 한 날짜에 일정이 3개 있을 때, **When** 주간 미리보기를 확인하면, **Then** 일정 2개와 `+1개` 더보기 슬롯이 표시된다.
3. **Given** 한 날짜에 일정이 8개 있을 때, **When** 주간 미리보기를 확인하면, **Then** 일정 2개와 `+6개` 더보기 슬롯이 표시된다.
4. **Given** 더보기 슬롯이 표시된 날짜가 있을 때, **When** 사용자가 날짜 칸을 선택하면, **Then** 기존 날짜 선택과 해당 일자의 상세 일정 표시 흐름이 유지된다.

### Edge Cases

- 일정 제목이 빈 문자열이더라도 슬롯은 지정된 한 줄 높이를 유지하고 인접 칸을 침범하지 않는다.
- 한 날짜의 첫 두 일정 제목이 모두 매우 길어도 각 슬롯은 서로의 세로 위치를 밀지 않는다.
- 검색 또는 참가자 필터로 일정 강조도가 낮아져도 크기와 오버플로 경계는 변하지 않는다.
- 오늘 표시와 선택 표시가 동시에 적용되어도 날짜 셀의 너비는 7열 배치를 벗어나지 않는다.
- 일정이 100개인 날짜도 미리보기에는 일정 2개와 정확한 초과 개수 슬롯 하나만 표시된다.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 주간 캘린더는 각 날짜의 일정 제목을 해당 날짜 칸 안에서 한 줄로 표시해야 한다.
- **FR-002**: 날짜 칸의 폭보다 긴 일정 제목은 인접 날짜 칸이나 다음 줄로 확장되지 않고 생략되어야 한다.
- **FR-003**: 주간 캘린더는 날짜별 일정을 최대 2개까지 표시해야 한다.
- **FR-004**: 날짜별 일정이 3개 이상이면 시스템은 `전체 일정 수 - 2`를 나타내는 더보기 슬롯 하나를 표시해야 한다.
- **FR-005**: 더보기 슬롯은 일정 슬롯과 동일하게 날짜 칸 안의 한 줄 높이와 폭 경계를 유지해야 한다.
- **FR-006**: 제목 및 더보기 표시 규칙은 현재 주, 이전 주, 다음 주 패널에 동일하게 적용되어야 한다.
- **FR-007**: 검색, 참가자 필터, 주간 이동, 날짜 선택 동작은 변경 전과 동일하게 유지되어야 한다.
- **FR-008**: 월간 캘린더와 선택 날짜 시간표의 일정 표시 방식은 변경되지 않아야 한다.
- **FR-009**: 일정 수 0, 1, 2, 3 및 다수에 대한 표시 개수 규칙은 반복 실행 가능한 자동 검증으로 보호되어야 한다.
- **FR-010**: 긴 한글, 긴 영문, 공백 없는 문자열의 날짜 칸 경계 준수 여부는 지정된 모바일 화면 너비에서 검증되어야 한다.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 320px, 375px, 430px 화면 너비에서 준비된 긴 제목 사례의 100%가 인접 날짜 칸 침범과 두 번째 줄 생성 없이 표시된다.
- **SC-002**: 일정 수 0, 1, 2, 3, 8, 100개 사례 모두에서 표시 일정 수와 더보기 개수가 정의된 규칙과 정확히 일치한다.
- **SC-003**: 현재 주, 이전 주, 다음 주의 준비된 모든 긴 제목 사례에서 동일한 한 줄 경계 규칙이 적용된다.
- **SC-004**: 검색, 필터, 주간 이동, 날짜 선택에 대한 기존 확인 시나리오에서 회귀가 0건이다.
- **SC-005**: 월간 캘린더와 선택 날짜 시간표의 기존 일정 표시 확인 시나리오에서 시각적 또는 기능적 회귀가 0건이다.

## Assumptions

- 주간 미리보기의 일정 순서는 현재 전달되는 일정 배열의 순서를 유지한다.
- 더보기 슬롯에는 별도 펼치기 동작을 추가하지 않고 기존 날짜 칸 선택 동작을 사용한다.
- 일정 제목의 전체 내용은 상세 일정 화면에서 확인할 수 있으므로 주간 미리보기에서는 한 줄 생략이 허용된다.
- 이번 변경은 표시 레이아웃과 표시 개수 검증에 한정되며 일정 데이터, 권한, 저장 방식은 변경하지 않는다.
- 월간 캘린더와 선택 날짜 시간표는 수정 범위 밖이며 회귀 확인 대상으로만 다룬다.
